Конструктор корабля. 
Использовался локально для партий D&D моих игроков. Возможно,кому-то также понадобится.

## Описание

Корабль собирается из модулей на клеточной сетке корпуса: модули покупаются на
верфи, складываются в инвентарь, устанавливаются на сетку перетаскиванием,
двигаются, поворачиваются и снимаются. Корабль автоматически пересчитывает ХП,
КД, скорость, нагрузку, энергию (МЭ), крен и орудийные лимиты, а панель ошибок
показывает, готов ли корабль к плаванию. 



## Запуск

Для обычного пользователя:

1. Откройте `open-ship-builder.html` двойным кликом.
2. Если браузер не сделал переход автоматически, откройте `dist/index.html`.

Проект уже собирается с относительными путями, поэтому статическая версия работает из файловой системы без локального сервера.

Для разработки:

```bash
npm install      # установка зависимостей
npm run dev      # запуск дев-сервера (http://localhost:5173)
```

Сборка статической версии (работает из файловой системы):

```bash
npm run build    # результат в папке dist/
npm run preview  # локальный предпросмотр сборки
```

Проверка типов: `npm run typecheck`.

## Как пользоваться

1. **Корабль** — главный экран: статус слева, сетка корпуса в центре, инвентарь и
   карточка выбранного модуля справа, ниже — ошибки сборки и журнал событий.
2. **Верфь** — нажмите «Обновить предложения»: верфь сгенерирует 1–6 случайных
   товаров (легендарные не появляются). Покупка списывает золото и кладёт модуль в
   инвентарь. Здесь же покупаются расширения корпуса (длина / ширина / этажность). 
3. Перетащите модуль из инвентаря на сетку. Подходящие клетки подсвечиваются
   зелёным, неверный тип — оранжевым, занятые/выход за границы — красным.
4. Кликните на модуль (в сетке или инвентаре), чтобы увидеть подробности, повернуть
   его, снять или вернуть в инвентарь.
5. **Редактор данных** — мастерский инструмент: правка базовых параметров корабля,
   ресурсов и всего каталога модулей (поиск, фильтры, добавление/удаление,
   импорт/экспорт каталога).
6. **Режим мастера** (галочка в шапке) открывает удаление модулей и редактирование ресурсов корабля. 
7. Кнопка огня дополнительно считает урон от каждого определенного снаряда на палубе, не дает стрелять если есть какая-то ошибка или некорректно собраны модули.
8. Реализована покупка припасов(продовольствия и воды) ведущая по формуле 1 игрок = 1 продовольствие и 1 вода 
При использовании кнопки окончания дня, продовольствие автоматически снимается 

### Сохранения
- Всё работает **локально без сервера**, данные хранятся в `localStorage`.
- Состояние автоматически сохраняется после каждого изменения.
- При запуске автоматически загружается последнее сохранение.
- Кнопки в шапке: ручное сохранение, продолжить, экспорт/импорт JSON,
  новая игра, сброс корабля, полный сброс (с подтверждением).
- Индикатор статуса: «Сохранено» / «Есть несохранённые изменения» / «Ошибка».
- Повреждённый импортируемый JSON не ломает приложение — показывается ошибка.

## Стартовый корабль

Сетка 4×4 (16 клеток), ХП 300, КД 15, скорость 40 фт, макс. нагрузка 24,
орудийный лимит 6 / тяжёлых 1 / сверхтяжёлых 0, макс. генератор уровня II.
Стартовые ресурсы: 30 000 зм и базовые припасы.

> Пустой стартовый корабль показывает ошибки «нет двигателя / киля / управления» —
> это нормально: их нужно устранить, установив соответствующие модули.


## Технологии

- **React 18** + **TypeScript**
- **Vite 5**
- **TailwindCSS 3**
- Хранение состояния в **localStorage** (автосохранение)
- Нативный **drag-and-drop** HTML5
- **Импорт/экспорт JSON** (полное состояние и каталог)

## Требования

- **Node.js** ≥ 18 (рекомендуется 20+; проверено на 24.16.0)
- **npm** ≥ 9 (проверено на 11.13.0)
- Современный браузер с поддержкой ES2020, `localStorage` и HTML5 drag-and-drop
- Сервер не требуется — приложение полностью клиентское

Полный список зависимостей с версиями — в [`requirements.txt`](requirements.txt)
и [`package.json`](package.json). Это проект на Node.js, поэтому зависимости
устанавливаются через `npm install`, а не через pip.
## Структура проекта

```
src/
  types.ts                 типы данных
  utils.ts                 утилиты (uid, форматирование)
  data/
    catalog.ts             полный каталог модулей (орудия, генераторы, …)
    defaults.ts            стартовое состояние, ресурсы, константы
  logic/
    grid.ts                генерация сетки корпуса по уровням расширений
    requirements.ts        разбор требований (генератор, ширина, …)
    compute.ts             расчёт статов, валидация, ошибки сборки, крен, энергия
    shipyard.ts            генерация предложений верфи
  state/
    persistence.ts         localStorage, импорт/экспорт, нормализация
    store.tsx              reducer + контекст + автосохранение
  components/              панели интерфейса (статус, сетка, верфь, редактор, …)
  ui/styles.ts             цвета категорий, редкостей, типов клеток
```

Ключ сохранения в `localStorage`: `darkwater-ship-builder:v1`.
